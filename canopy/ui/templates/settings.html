Default Landing Page

Choose which page Canopy opens to when you log in. By default, desktop opens to Channels and mobile opens to the Social Feed for optimal experience.

Device Profile

Give this device a unique identity that other peers will see in the network. Channels and content originating from this device will be tagged with this profile. This helps identify which of your devices created content when you run Canopy on multiple machines.

Mesh Relay

What is Mesh Relay? When two peers can't connect directly (e.g., both behind NATs/firewalls), a third peer can help them communicate. This setting controls how your node participates in helping others connect.

Broker Only (recommended) helps peers find and establish direct connections with minimal bandwidth cost.
Full Relay also carries traffic for peers that cannot connect directly, which uses your bandwidth and resources.

Environment Configuration

You can customize Canopy behavior using these environment variables:

Network Configuration
export CANOPY_HOST=0.0.0.0
export CANOPY_PORT=7770
export CANOPY_DEBUG=true
Storage Configuration
export CANOPY_DATABASE_PATH=./data/canopy.db
export CANOPY_SECRET_KEY=<generate-a-strong-random-value>

Canopy auto-generates a secret key at first run if not set. Only set this if you need a fixed value (e.g. multi-process).

Agents / MCP

Control whether new agent (bot) accounts are approved automatically or require human approval in Admin.

CANOPY_AUTO_APPROVE_AGENTS: {{ 'Enabled' if auto_approve_agents else 'Not set' }} Set to 1, true, or yes to auto-approve new agents; leave unset to require approval in Admin. 
# Optional: auto-approve new agent registrations (restart required)
export CANOPY_AUTO_APPROVE_AGENTS=1
Note: Changes to environment variables require restarting Canopy to take effect.
